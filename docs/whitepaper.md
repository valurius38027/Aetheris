## 1. 摘要 (Abstract)

Aetheris 是一种新型的隐私加密货币，旨在解决传统公链在面对非理性恶意攻击者时的脆弱性。通过抛弃全局共识依赖，转而采用客户端侧验证（Client-Side Validation）、递归零知识证明（Recursive ZK-SNARKs）以及基于可验证延迟函数（VDF）的时间发行机制，Aetheris 在数学上保证了资金的完全隐匿性与系统的极高容错性。

## 2. 核心哲学：主权验证与创世锚点

Aetheris 认为在极端对抗环境下，传统“诚实多数派”假设是危险的。

- **第一公理：创世锚点 (Genesis Anchor)**：协议认定创世交易（Genesis Transaction）及其哈希值为绝对可信的数学原点。
- **第二公理：形式化安全性 (Formalized Security)**：Aetheris 拒绝"代码即法律"的模糊性，追求"数学证明即法律"。所有核心协议逻辑、密码学谓词及 ZK-VM 指令集必须经过形式化验证，以确保在面对非理性攻击者时具备机器可校验的安全性。
- **第三公理：主权验证 (Sovereign Verification)**：每个用户的客户端均以创世锚点为逻辑起点进行独立验证。客户端不依赖网络多数的"共识"来判断状态真伪，而是通过本地验证 VDF 链、ZK 证明和 Nullifier 唯一性来确定事实。这构成了对日蚀攻击的数学根本性防御——即使完全与诚实网络隔离，节点也无法被欺骗接受非法状态，且可在重新连接后通过本地 VDF 重算恢复规范链。

## 3. 隐私架构：暗池执行

### 3.1 递归 ZK-SNARKs (Halo 2 / Plonky2)
所有交易均通过递归零知识证明进行封装。每一笔新的交易证明都会包含前一状态的合法性证明，从而实现：
- **资金链切断**：在数学上无法关联输入 UTXO 与输出 UTXO。
- **极速验证**：客户端只需验证最新的递归证明，即可确认整个账本历史的合法性。

### 3.2 全同态加密 (FHE) 与主权 ZK-合约
Aetheris 支持基于 ZK-VM 的隐私智能合约。
- **本地执行**：合约逻辑在用户本地运行，仅向网络提交状态转换的 ZK 证明。
- **记录模型 (Record Model)**：通过加密的状态记录实现并发处理，确保复杂逻辑下依然保持极高的隐私性与抗审查性。

## 4. 发行与共识：VDF 时间驱动与数学仲裁

Aetheris 不依赖工作量证明（PoW）、权益证明（PoS）或伪随机抽签（VRF）。共识分两阶段运行：Gossip 分发阶段与数学仲裁阶段。

- **Proof of Time (PoT)**：利用连续、不可并行的 VDF（可验证延迟函数）计算作为发行与区块提议的唯一依据。每个节点独立计算 VDF 序列，不依赖网络输入。
- **物理代价公平性**：VDF 确保了无论攻击者拥有多少并行算力，其生成新币与区块提议的速度受到单核物理主频的绝对限制。这种"算力无法换取时间"的特性，需作精确解读：**并行算力不能加速 VDF**（VDF 是顺序构造），但**单核性能提升会线性增加出块速率**（单核 2x 快 ≈ 2x 出块）。这一线性关系与 PoW 的并行指数级优势有本质区别，但仍需要**难度自平衡机制**来维持恒定出块时间。
- **Gossip 分发阶段**：节点在完成 VDF 迭代后，通过 P2P 网络广播其区块提议（包含 VDF 结果、证明及待打包交易）。此阶段需要网络通信，但通信内容仅包含候选提议——不含投票或共识消息。
- **数学仲裁 (Mathematical Arbitration)**：节点收到来自不同对等节点的多个区块提议后，在本地通过确定性函数选择胜者：

  ```python
  winner = min_by_key(proposals, key = blake3(prev_block_hash ∥ vdf_result))
  ```

  此函数满足以下数学性质：
  - **单调性**：向集合中添加新提议不会改变最小值（若更小者已存在）。
  - **序无关性**：提议到达顺序不影响胜者选择。
  - **局部确定性**：给定相同的提议集合，任意节点独立计算均得同一胜者。

  上述特性保证了仲裁步骤本身**无须显式沟通**——节点无需投票、轮次协调或超时同步。仲裁的收敛性完全由排序函数的数学性质保证，而非网络协议。

- **隐匿发行**：新币产生的奖励通过零知识证明直接注入屏蔽池（Shielded Pool），确保初始代币分配的匿名性。

### 4.1 VDF 链作为时间真相源

VDF 序列 `y_0, y_1, ..., y_n` 完全由创世种子 `S` 顺序确定。此序列构成**数学时间轴**——每个节点自创世以来所经历的确切迭代次数是可独立计算的。区块提议附属于此时间轴：提议的合法性通过 VDF 结果 `y_i` 验证，而非通过网络接收顺序。这从根本上消除了日蚀攻击者通过隔离来重写时间线的可能性。

### 4.2 VDF 难度自平衡 (Difficulty Self-Balancing)

单核性能随硬件代际提升，若 VDF 难度固定，出块率将持续加速。Aetheris 通过**确定性难度重定向**维持恒定出块时间。

**重定向公式**（与比特币难度调整相同结构）：

$$D_{n+1} = D_n \times \frac{T_{target} \times N}{\Delta t}$$

其中：
- $D_n$ = 当前难度（VDF 迭代次数）
- $T_{target}$ = 目标出块时间（10 秒）
- $N$ = 重定向窗口（10 个区块）
- $\Delta t$ = 窗口内 $N$ 个区块的总实际时间（基于区块时间戳）

**自强制执行约束**：

此公式的生效并非依赖节点"遵守规则"，而是由密码学约束自动强制：

1. **VDF 证明绑定难度**：Wesolowski 证明的验证方程中显式包含 `difficulty` 参数：
   $$y = \pi^l \cdot x^{2^{difficulty} \pmod l} \pmod{modulus}$$
   若区块声称 `difficulty = D` 但实际计算量不同，验证方程不成立。

2. **预期难度是确定性的**：每个高度 $h$ 的预期难度通过链上数据独立计算：
   $$D_h = \begin{cases} D_{genesis}, & h < N \\ retarget(chain_{h-N}, ..., chain_{h-1}), & h \geq N \end{cases}$$
   节点在收到区块时验证 `block.difficulty == D_h`，不匹配则拒绝。

3. **时间戳操纵限制**：攻击者若伪造时间戳来降低难度，将面临两难：
   - 假造更早时间戳（增大 $\Delta t$）→ 难度降低，但违反因果律（时间戳不能小于父区块时间戳）
   - 假造更晚时间戳（减小 $\Delta t$）→ 难度升高，攻击者自我惩罚
   - 最大偏移限制在 2 小时内，在 $N=10$ 的窗口上最大难度误差约 2%，下一轮自动修正

4. **分叉惩罚**：若攻击者使用错误难度生成区块，VDF 结果不同，导致后续 VDF 种子链分叉。维持分叉需要真实的顺序 VDF 计算，代价随高度线性增长。

**硬件速度与功率的关系**：
- $D_{genesis} = 1{,}600{,}000$ 迭代（2026 年参考值）
- 目标出块时间 $T_{target} = 10$ 秒
- 等效单核速度：约 160K 平方/秒 ≈ 单核 CPU 的典型 RSA-2048 模幂运算吞吐
- 当硬件速度翻倍时，难度将在 $N$ 个区块内自动翻倍，恢复目标出块时间

## 5. 经济模型：公平线性排放

### 5.1 核心理念

Aetheris 的经济模型与其密码学哲学一致——**数学约束驱动，经济激励最小化**：

- **Fair Launch**：零预挖、零创始人分配、零基金会资金。所有 AET 通过 VDF 挖矿产出。
- **线性排放**：区块奖励从 1 AET 线性递减至零，总供给约 21,000,000 AET，排放周期约 13.3 年。
- **手续费燃烧**：交易手续费 100% 销毁，仅作为防 spam 机制，不激励矿工。
- **无 Staking**：网络安全依赖 VDF 串行性和 ZK 正确性，而非经济质押。

此模型与白皮书 §6.1 的安全哲学一致——"Aetheris 的安全性建立在数学约束而非经济激励之上"。

### 5.2 排放公式

$$\text{reward}(h) = \max\left(0,\ R_0 \times \left(1 - \frac{h}{N}\right)\right)$$

| 参数 | 值 | 含义 |
|------|-----|------|
| $R_0$ | 1 AET (10^8 atoms) | 初始区块奖励 |
| $N$ | 42,000,000 区块 | 排放总区块数 |
| $T_{block}$ | 10 秒 | 目标出块时间 |
| $N \times T_{block}$ | ~13.3 年 | 总排放周期 |
| 总供给 | $\frac{R_0 \times N}{2} = 21,000,000$ AET | 精确数学上限 |

**逐年排放示意**：

```
第 1 年 (~3.15M 区块):  ~3.0M AET (14.3%)
第 2 年 (~3.15M 区块):  ~2.7M AET (12.9%)
```

### 5.3 创世区块

创世区块（高度 0）不包含任何预分配交易。区块链从空状态启动，第一个矿工通过 VDF 挖矿获得高度 0 的区块奖励（1 AET）。  

- **无 Genesis Mint**：无创始人大额铸造。
- **无 Genesis Transfer**：无开发/基金会转账。
- **无冻结地址**：不存在特权地址。
- **创世哈希**：空区块的确定性哈希。

### 5.4 交易手续费

$$\text{fee} = \max(\text{MIN\_TX\_FEE},\ \text{tx\_size} \times \text{FEE\_PER\_BYTE})$$

| 参数 | 值 | 含义 |
|------|-----|------|
| MIN_TX_FEE | 0.0001 AET (10,000 atoms) | 最小手续费 |
| FEE_PER_BYTE | 0.000001 AET (100 atoms) | 每字节费率 |
| 手续费去向 | 100% 燃烧 | 永久退出流通 |

**燃烧机制**：区块总手续费从该区块的 coinbase 奖励中扣除：
$$\text{coinbase} = \max(0,\ \text{block\_reward} - \sum \text{tx\_fees})$$

手续费通过 `public_amount` 字段在 ZK 守恒电路中强制约束：
$$\text{total\_in} - \text{total\_out} - \text{public\_amount} = 0$$

其中 `public_amount > 0` 代表手续费。此机制不改变现有 ZK 电路结构。

## 6. 网络层安全性：恒定熵路由与日蚀抵抗

为了对抗流量分析和针对性日蚀攻击，Aetheris 强制集成网络层混淆。日蚀攻击的威胁模型是：攻击者垄断目标节点与诚实网络间的所有通信路径，可选择性地扣留区块提议或注入虚假数据。Aetheris 对此采用分层防御。

### 5.1 数学层防御（根本保证）

以下保证由协议的密码学核心提供，**不依赖网络层配置**：

- **完整性保证**：即使被完全隔离，节点也无法被欺骗接受非法区块。每个区块必须通过 VDF 证明验证、ZK 聚合证明验证、Nullifier 唯一性检查及创世锚定校验。上述验证均是本地计算，不依赖外部输入。
- **可检测性保证**：VDF 时间轴是确定性的。若节点本地已计算 n 个 VDF 迭代但仅收到 m < n 个对应区块，则节点数学上确定自己被日蚀攻击。此检测非概率推测，而是 `n ≠ m` 的布尔事实。
- **可恢复性保证**：重新连接后，节点通过本地重算 VDF 序列确定规范链。VDF 是确定性的——连接前后的计算结果严格一致。"追赶"是本地验证而非信任行为。
- **有限损害保证**：日蚀攻击者不能窃取代币（需 ZK 密钥）、不能对诚实网络双花（Nullifier 唯一性）、不能造成永久分叉（VDF 链选择）。攻击者维持假链的代价受限于单核 VDF 计算速度：每单位时间仅能产生一步 VDF，无法并行加速。

### 5.2 网络层防御（混淆与扩散）

为了降低节点被针对性隔离的概率：

- **Mixnet (Loopix)**：所有数据包在传输过程中经过多层随机延迟和重新排序，使攻击者难以定位目标节点的流量模式。
- **Cover Traffic (覆盖流量)**：无论是否有实际交易，客户端均以恒定速率发送加密混淆包，消除流量分析的时间侧信道。
- **Sphinx 数据包格式**：确保每一跳路由节点仅能感知局部拓扑，无法重构全局传输路径。

## 7. 容错与安全性

### 7.1 对抗非理性攻击
系统不预设攻击者是“追求经济利益的”。Aetheris 的安全性建立在数学约束而非经济激励之上。

**双花防御**：通过 Nullifiers（作废凭证）的数学唯一性，系统在逻辑上禁止了双花。即使攻击者控制了目标节点的所有网络连接（日蚀攻击），也无法在诚实的全局网络中完成双花——Nullifier 的全局唯一性由每笔交易的 ZK 证明强制执行。

**恶意区块扣留防御**：恶意节点即使拒绝打包合法交易，也无法伪造非法交易，因为它们无法生成有效的 ZK-Proof。此外，VDF 的确定性保证了规范链的持续延伸——即使所有区块提议均被扣留，节点仍能独立计算当前 VDF 状态，并在重新连接后通过 VDF 重算验证诚实链。

**容错界限**：系统在以下极限条件下仍保证安全性：
- 当节点被完全日蚀隔离时：资金安全、状态不会被篡改、最终可恢复。
- 当攻击者控制超大规模算力时：无法窃取非其所有的资金（需 ZK 密钥），无法制造虚假状态（需有效 ZK 证明），无法加速发行（VDF 是顺序的）。
- 当攻击者不追求经济理性时：系统安全性由数学约束保证，不依赖"攻击者将损失金钱"的假设。

### 7.2 后量子抵抗
采用基于格（Lattice-based）的加密算法，确保系统在量子计算时代依然具备数学上的陷门安全性。
